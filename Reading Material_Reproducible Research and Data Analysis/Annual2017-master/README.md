    <article class="markdown-body entry-content" itemprop="text"><h1><a href="#intro-to-pandoc" aria-hidden="true" class="anchor" id="user-content-intro-to-pandoc"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Intro to Pandoc</h1>
<h2><a href="#garret-christensen" aria-hidden="true" class="anchor" id="user-content-garret-christensen"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Garret Christensen</h2>
<p><a href="http://pandoc.org/" rel="nofollow">Pandoc</a> is a command line tool to convert documents from one type to another. Kind of like <a href="https://youtu.be/wGKxWatPkd0" rel="nofollow">Professor Farnsworth</a>.
<a href="https://camo.githubusercontent.com/c87b9b9af3f3e4dcc9ab8f724729208283a966a9/687474703a2f2f7777772e6675747572616d612d6d6164686f7573652e6e65742f62696f732f62696f4661726e73776f7274682e6a7067" target="_blank"><img src="https://camo.githubusercontent.com/c87b9b9af3f3e4dcc9ab8f724729208283a966a9/687474703a2f2f7777772e6675747572616d612d6d6164686f7573652e6e65742f62696f732f62696f4661726e73776f7274682e6a7067" alt="" data-canonical-src="http://www.futurama-madhouse.net/bios/bioFarnsworth.jpg" style="max-width:100%;"></a></p>
<h3><a href="#installing-pandoc" aria-hidden="true" class="anchor" id="user-content-installing-pandoc"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installing Pandoc</h3>
<p>Pandoc is included when you install RStudio, and likely other things too (like the Anaconda installation of Python).
To make sure you have it, type <code>which pandoc</code> or <code>pandoc --version</code> in the command line. If it's not installed, download it <a href="https://pandoc.org/installing.html" rel="nofollow">here</a>.</p>
<h3><a href="#using-the-command-line" aria-hidden="true" class="anchor" id="user-content-using-the-command-line"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Using the Command Line</h3>
<p>The command line is the way to type commands directly to your computer. On a Mac, look in ./Applications/Utilities/Terminal. Windows usually comes with 'Powershell' installed, but I like <a href="https://git-scm.com/download/win" rel="nofollow">Git Bash</a>.
If you're using Linux, I'm going to assume you know what you're doing.</p>
<p>We'll go over some basic commands, a more complete list is <a href="https://lifehacker.com/5633909/who-needs-a-mouse-learn-to-use-the-command-line-for-almost-anything" rel="nofollow">here</a>. Software Carpentry has a nice tutorial <a href="http://swcarpentry.github.io/shell-novice/" rel="nofollow">here</a>.</p>
<p>Just a few of the very basics:</p>
<ul>
<li><code>whoami</code> to check the current user</li>
<li><code>cd</code> to change directory. Follow with <code>directoryname</code> to go down into a directory, or <code>..</code> to go up a level</li>
<li><code>pwd</code> to see the present working directory</li>
<li><code>ls</code> lists the contents of the current directory.</li>
</ul>
<h3><a href="#reproducibility" aria-hidden="true" class="anchor" id="user-content-reproducibility"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Reproducibility</h3>
<p>What does this have to do with transparency or reproducibility?</p>
<p>It can help you automate stuff! If you routinely produce a report, and want to put it on the web, or you wrote in LaTeX and a journal doesn't accept that so you need to submit in Word, or if you hate the Microsoft Word Equation Editor (because who doesn't?) Pandoc is your friend.</p>
<h3><a href="#simple-conversion" aria-hidden="true" class="anchor" id="user-content-simple-conversion"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Simple Conversion</h3>
<p><code>pandoc --help</code> to see all the options. The general setup is <code>pandoc &lt;existingfile.type&gt; &lt;newfile.type&gt;</code>.</p>
<h4><a href="#change-output-format" aria-hidden="true" class="anchor" id="user-content-change-output-format"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Change Output Format</h4>
<p>By default output is html.</p>
<p>So type <code>pandoc militaryrecruiting.tex</code>. But this just outputs the file to the command line.</p>
<p>To save the output to a new html file: <code>pandoc militaryrecruiting.tex -o militaryrecruiting.html</code></p>
<p>(The easiest way to open any of your output files is just type <code>open &lt;filename&gt;</code>.)</p>
<p>Just change the extension to get whatever output file you like.
<code>pandoc militaryrecruiting.tex -o militaryrecruiting.docx</code></p>
<p><code>pandoc militaryrecruiting.tex -o militaryrecruiting.odt</code></p>
<p>You'll notice some labels and references don't look so hot immediately, but footnotes are great, and a lot (but not all) of the math is nice.</p>
<p>To try Markdown, try converting the README file to html. To go from MS Word to LaTeX, send the Word paper back to TeX. How are the original and re-converted different?</p>
<h4><a href="#references" aria-hidden="true" class="anchor" id="user-content-references"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>References</h4>
<p>To get the references, tell pandoc where the .bib file is.
<code>pandoc militaryrecruiting.tex -o militaryrecruiting.docx --bibliography=military_bib.bib --csl=chicago-author-date.csl</code></p>
<p>Note that you also have to tell it about the citation style file.</p>
<h4><a href="#make-a-script" aria-hidden="true" class="anchor" id="user-content-make-a-script"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Make a Script</h4>
<p>If you wanted to really automate this, you could make the conversion part of a shell script. Put the command into a text file, which you can do directly, or with the <code>echo</code> command:</p>
<p><code>echo "pandoc militaryrecruiting.tex -o militaryrecruiting.docx --bibliography=military_bib.bib --csl=chicago-author-date.csl"&gt;&gt;shellscript.sh</code></p>
<p>Then you can run that script by typing:</p>
<p><code>bash shellscript.sh</code></p>
<p>Or you could run that in Stata with:</p>
<p><code>!bash shellscript.sh</code></p>
<h3><a href="#issues" aria-hidden="true" class="anchor" id="user-content-issues"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Issues</h3>
<p>Do you have issues? Pandoc is open source and <a href="https://github.com/jgm/pandoc">on GitHub</a>.</p>
<p>I haven't been able to get complicated tables to work. That's potentially a big limitation.</p>
<p>Non-ASCII characters can screw things up. (Apparently, BibTeX doesn't support Unicode.) For example, try converting my "Manual of Best Practices" into HTML.</p>
<p><code>pandoc Manual.tex -o Manual.html --bibliography=test.bib</code></p>
<p>There definitely is a way to fix this!</p>
<p>One way would be to use <code>iconv</code> to clean up the .bib file. This should convert a UTF-8 file to ASCII. F is for from, t is for to, and c discards any unconvertable characters.</p>
<p><code>iconv -c -f utf-8 -t ascii test.bib&gt;clean.bib</code></p>
<p>(You could also try using <code>sed</code>, the stream editor: <code>sed -i 's/[\d128-\d255]//g' MYFILE.txt</code> to remove all the non-ASCII characters.)</p>
